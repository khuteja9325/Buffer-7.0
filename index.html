Attack Simulation Lab

Simulate real-world cyberattack scenarios. Each attack is detected by the Merkle Tree engine, demonstrating why O(log n) traversal is essential for large file systems.

Byte Flip
Flip a single byte in a random file. Even 1 changed byte cascades up the tree and changes the root hash.
Detection: O(log n)
💉
Malicious Injection
Inject a new unauthorized file into the directory. Unauthorized files change the tree structure and root hash.
Detection: O(log n)

File Deletion
Silently delete a file. The missing leaf node causes hash mismatches up the tree path.
Detection: O(log n)

Change only file timestamp (not content). Merkle Tree is content-based — metadata spoofing is correctly ignored.
Result: No alert (correct)
💣

Compromise 50% of files simultaneously (ransomware-style attack). Tree immediately detects all changes.
Detection: O(n log n) total

Attacker attempts to replace the trusted root hash. System detects the root mismatch immediately.
Detection: O(1)

What is a Merkle Tree?

A Merkle Tree is a complete binary tree where each leaf node contains the cryptographic hash of a data block (here, a file), and each internal node contains the hash of its two children concatenated. The root hash acts as a single fingerprint for all data.

Time Complexity

Build Tree
O(n)
Hash each file, combine level-by-level
Detect Tamper
O(log n)
DFS: skip clean subtrees immediately
Verify Proof
O(log n)
Only need sibling hashes up the path
Naive Scan
O(n)
Must check every file individually

Data Structures Used

Binary TreeCore Merkle Tree structure — each node has left/right children
Hash MapMaps filenames → baseline hashes for O(1) lookup
Queue (BFS)Tree rendering traverses level-by-level using BFS
Stack (DFS)Tamper detection uses recursive DFS to locate corrupted subtrees
ArrayLeaf nodes temporarily stored before bottom-up construction
